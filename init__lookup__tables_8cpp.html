Detailed Description

Functions translating lists build from the infile into lookup_tables.

Author
Bastian Knippschild
Markus Werner

The lookup tables only contain unique quantum number combinations and from the lists and latter are replaced by indexlists referring to those lookup tables.

This automatically avoids recalculation of operators or even complete correlators

Build an array with all the quantum numbers needed for a particular correlation function respecting physical conservation laws

Parameters
[in]correlatorA single correlator specified in the infile and processed into the Correlators struct
[in]operator_listList of all operators specified in the infile and processed into Operators struct
[out]quantum_numbersA list of all physical quantum numbers as specified in the QuantumNumbers struct that are possible for correlator

correlator contains multiple operator_numbers. From combinatorics a large number of combinations arise. In general only a subset of them are physically allowed or necessary to calculate. In this function momentum conservation is enforced and multiple cutoffs introduced.

Restriction to what shall actually be computed is done in if statements for each diagram because it depends on the number of quarks.

Bug:
In build_Q1_lookup the order of quarks given is consistently switched.
  1. Build an array (quantum_numbers) with all the quantum numbers needed for this particular correlation function.
  2. Build the lookuptable for VdaggerV and return an array of indices corresponding to quantum_numbers computed in step 1. In vdv_indices the first entry is the id of vdv, the second tells us if vdv must be daggered to get the correct quantum numbers.
  3. Build the lookuptable for rVdaggerVr and return an array of indices corresponding to the 'quantum_numbers' computed in step 1.
  4. Build the lookuptable for Q2 and return an array of indices corresponding to the 'quantum_numbers' computed in step 1.
  5. Build the lookuptable for the correlation functions
  6. Repeat steps 1.-5. for all correlators in correlator_list. Where applicable rVdaggerVr must be replaced by rVdaggerV in step 3. and Q2 by Q1 in step 4.

Sets index_of_unity to the index of operator_lookuptable.vdaggerv_lookup where momentum and displacement are both zero, or to -1 if no such entry is found.
