Detailed Description

Functions translating lists build from the infile into lookup_tables.

The lookup tables only contain unique quantum number combinations and from the lists and latter are replaced by indexlists referring to those lookup tables.

This automatically avoids recalculation of operators or even complete correlators

Build an array with all the quantum numbers needed for a particular correlation function respecting physical conservation laws.

Parameters
[in]correlatorA single correlator specified in the infile and processed into the Correlators struct
[in]operator_listList of all operators specified in the infile and processed into Operators struct
[out]quantum_numbersA list of all physical quantum numbers as specified in the QuantumNumbers struct that are possible for correlator
[in]momentum_cutoffCutoffs for sum of momenta squared, see GlobalData::momentum_cutoff.

correlator contains multiple operator_numbers. From combinatorics a large number of combinations arise. In general only a subset of them are physically allowed or necessary to calculate. In this function momentum conservation is enforced and multiple cutoffs introduced.
